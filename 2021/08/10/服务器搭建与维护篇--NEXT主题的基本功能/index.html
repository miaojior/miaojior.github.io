服务器搭建与维护篇--NEXT主题的基本美化

发表于 2021-08-10 阅读次数： 本文字数： 6k

本教程是以举例ymmc如何建成为主，希望你不只是复制命令，而是理解目录之间的存储关系，举一反三，建立自己的博客。看这篇文章之前，希望你已经使用hexo发表过文章。

  • 注：本博客非C++代码中 /**/ 的注释格式(一般在一个代码块开头)表示接下来这段内容来自的文件。所有目录都是对于hexo根目录而言的。
  • 注：配置yml配置文件时配置项冒号后和配置信息之间要保留一个空格。 如果在yml文件中出现#则代表本行其后内容是注释，系统会直接跳过#后内容 （因此如果你想关闭一个功能能，不要删除其配置信息，直接没一行在前面打上#即可，方便再次启用）
  • 注：配置文件采用的都是Python式的缩进，行前缩进数量相同的为同级配置，缩进少的比缩进多的配置级别高。
  • 注：本文将直接给出配置文件截取部分，与源文件不同的更改将以注释的方式展示，当然你也可以通过关于页面直接下载网站的源码（源码中不包含本文的修改注释）

① 选择主题并允许开启darkmod

/*themes/next/_config.yml*/
# Schemes
scheme: Muse  #可用值: Muse | Mist | Pisces | Gemini
# Dark Mode
darkmode: true
  • YMMC选择了默认主题，如果你喜欢其他主题，请删掉其前面的# 并恢复弃用主题行前#
    关于主题样式的演示，请自行以换主题的方式欣赏并选择
  • 当drakmode值为true时next主题会根据系统/浏览器设置自行开启关闭darkmode

② 网站信息设置

配置网站图标

favicon:
  small: /images/ymmc_logo.svg #小图标 参考大小16×16
  medium: /images/ymmc_logo.svg #大图标 参考大小32x32
  apple_touch_icon: /images/ymmc_logo.svg #apple移动设备 参考大小180x180
  safari_pinned_tab: /images/ymmc_logo.svg #safari固定选项卡 参考大小矢量图

如果你使用位图建议你配置为不同大小的图标配置不同的图片，参考大小来源是next配置的官方图标的大小。不过YMMC的尝试已经证实全部使用矢量图完全没有问题。

配置开源许可协议

Creative Commons 4.0是一种知识共享协议，支持6种不同许可方式和39种语言
详情见https://creativecommons.org/about/cclicenses/

/*themes/next/_config.yml*/
creative_commons:
license: by-nc-sa  #可用值: by | by-nc | by-nc-nd | by-nc-sa | by-nd | by-sa | cc-zero
size: small  #可用值: big | small
sidebar: true
post: true
language: deed.zh
  • license 许可证书，必须是6种可用值之一，关于证书详细信息见CC官网
  • size 证书图标大小，只能是big/small，请自行配置博客查看效果，并选择大小
  • sidebar 值为true时，将在侧栏显示证书
  • post 值为true时，将在发表的文章下方显示该版权许可
  • language 证书的语言，你可以选择 deed.zh（中文）/deed.en（英文）,或者到CC官网选择对应语言的证书。对应页面完整网址最后一个“/”后的部分就是要填入此行的部分
/*_config.yml*/
url: http://39.106.182.159/
  • 注：此配置项在工作目录下的_config.yml中
  • 此项必须配置，内容为网站的IP地址或域名，否则本文链接无法正确链接

由于一些懂得都懂的原因 cornmc.top 的域名还不能访问服务器的80端口 所以此处YMMC使用了IP

③ 配置侧栏功能

sidebar:
  position: left  #可用值: left | right
  #width: 300
  display: hide  #可用值: post | always | hide | remove
  padding: 80
  offset: 12
  • position 侧栏显示位置 左侧或右侧
  • width 侧栏宽度 如果不进行设置（如像YMMC这样把它注释掉）Muse和Mist的默认宽度是320 Pisces和Gemini默认宽度是240
  • display侧栏显示方式
    • post 仅在发表的文章中显示
    • always 全站显示
    • hide 全站隐藏，由浏览者自行打开
    • remove 完全移除侧栏功能
  • padding 侧栏第一条信息距顶部的距离
  • offset 侧栏相对于顶栏的偏移量
    注：此参数仅在Pisces和Gemini中有效
avatar:
  url: /images/ymmc_logo_white-letters.svg
  rounded: false
  rotated: false
site_state: true
  • url 作者头像，你可以填入任何网址，YMMC这样填入的是本站的相对地址，需要提前把文件放到/source/images/
    你可能会好奇为什么这回不是ymmc_logo.svg了，因为ymmc_logo.svg这张图的字母是镂空的，六边形是黑色的，而next主题侧栏底色是深灰色，就看不清了。所以在使用svg是请注意修改图片底色
    svg是一种矢量图，他可以任意放大而不模糊，有时间可能会写一篇文章讲讲这张图是怎么写出来的。（是的不是画是写出来的）
  • rounded 值为true时，头像将会裁剪为原型
  • rotate 值为true时头像将会旋转
  • site_state 值为true 将会在侧栏显示网站状态，如文章数和标签数
social:
  #GitHub: https://github.com/yourname || fab fa-github
  #E-Mail: mailto:yourname@gmail.com || fa fa-envelope
  #Weibo: https://weibo.com/yourname || fab fa-weibo
  #Google: https://plus.google.com/yourname || fab fa-google                                      #Twitter: https://twitter.com/yourname || fab fa-twitter
  #FB Page: https://www.facebook.com/yourname || fab fa-facebook
  #StackOverflow: https://stackoverflow.com/yourname || fab fa-stack-overflow
  #YouTube: https://youtube.com/yourname || fab fa-youtube
  #Instagram: https://instagram.com/yourname || fab fa-instagram
  #Skype: skype:yourname?call|chat || fab fa-skype
social_icons:
  enable: true
  icons_only: false
  transition: false
  • social 你可以在此处留下你的社交媒体信息 格式是 社交媒体名词: 网址 || 图片
    同样的没有给出示例的图片你要到Font Awesome上找
  • enable 值为true时，将会显示对应的社交媒体图片
  • icons_only 值为true时，将会只显示社交媒体图片
  • transition 值为true将会开启过度效果
links_settings:
  icon: fas fa-user-friends
  title: 友情链接
  layout: inline #可用值:  block | inline
links:
  大连育明高级中学官方网站: http://www.dlymgz.cn
  Minecraft官网: http://minecraft.net
  myxr_crs的博客: http://myxrcrs.cn
  miaojior的博客: http://39.106.182.159:8000
  KHLFYY的博客: http://khlfyy.top

使用此功能你可以在侧栏中添加一些外链

  • icon 外链标题前的图标
    同样的没有给出示例的图片你要到Font Awesome上找
  • title 所有外链的总标题
  • layout 当有多个外链时的显示方式
    • block 竖向优先
    • inline 横向优先
  • links 请使用 显示名称: 网址 的格式填入所有外链，一行一个。
toc:
  enable: true
  number: true
  wrap: false
  expand_all: true
  max_depth: 6

使用此功能你可以在发布文章的侧栏中显示概要

  • enable 值为true时开启此功能
  • number 值为true时将在每个标题前显示x.x.x的章节号
  • wrap 值为true时当标题一行无法完全显示会自动换到第二行，值为false将显示为“好长的标题啊啊啊啊…”
  • expand_all 值为true将展开所有标题，值为false只展开现在正在阅读部分的标题
  • max_depth 标题最大展开深度，等级区分请看Markdown语法中的六级标题表示方法

④ 配置底部设置

footer:
  since: 2020
  icon:
    name: fa fa-heart
    animated: false
    color: "#ff0000"
  copyright: YMMC
  powered: true
  • since 网站开始运营时间
  • name 使用的图标的名称
    同样的没有给出示例的图片你要到Font Awesome上找
  • animated 当值为true时会为图标添加跳动效果
  • color 图标的填充颜色，颜色采用的是HTML颜色代码。
  • copyright 作者名称，如果不配置将默认使用hexo的_config.yml中的名称
  • powered 值为true时会显示 Hexo ＆ Next 强力驱动
beian:

⑤设置发布文章

excerpt_description: true
read_more_btn: true

excerpt_description 值为true时在主页显示文章description的内容,作为摘要，而不显示全文。这样可以有效的避免主页内容过多。
read_morn_btn 值为true时在摘要之后显示阅读更多按钮

文章的描述以和title相同的形式写在xxx.md的头部分如

---
title: 服务器搭建与维护篇--NEXT主题的基本美化
date: 2021-08-10 12:00:00
tags: [ '技术' , '后端' , '网站' ]
description: 在这里写文章的描述！！！

---

不过如果你只想摘取文章从头开始的一部分也可以通过在这部分之后加以下标志，只在主页显示它之前的文章内容。这无疑是一种更简单，更值得推荐的方式

<!-- more -->
  • 注意： 使用more截取文章的方式不需要关闭excerpt_description 但是如果你同时在文章中添加description和more 则主页只会显示description的内容
post_meta:
  created_at: true
  updated_at:
    enable: false
    another_day: true
  categories: false
  • created_at 值为true时将显示帖子的创建时间
  • update_at
    • enable 值为true时将显示帖子最后一次的修改时间
    • another_day 值为true时 只有当最后一次修改和创建的时间不在同一天时才会显示最后一次修改的时间
  • categories 值为true时显示文章分类的信息
tag_icon: true
  • tag_icon 值为true时将在文章底部的每个标签上显示标签图标，否则只显示文字。
reward_settings:
  enable: true
  animation: false
  comment: Buy me a coffee
reward:
  wechatpay: /images/miaojior_wechat.png
  alipay: /images/miaojior_alipay.jpg
  #paypal: /images/paypal.png
  #bitcoin: /images/bitcoin.png
  • enable 值为true时开启赞赏功能
  • animation 值为true时开启动画效果
  • comment 填入赞赏功能的注释
  • reward 添加收款码图片
follow_me:
  #Twitter: https://twitter.com/username || fab fa-twitter
  #Telegram: https://t.me/channel_name || fab fa-telegram
  #WeChat: /images/wechat_channel.jpg || fab fa-weixin
  #RSS: /atom.xml || fa fa-rss
  • flollow_me 你可以按照 社交媒体： 地址/图片 || 图标 的格式添加你的频道以便大家关注。
    没有给出示例的图片你可以到Font Awesome上找
post_navigation: false #可用值 left | right | false
  • post_navigation 在文章最后显示上一篇下一篇
    • left 左边上一篇，右边下一篇
    • right 左边下一篇右边上一篇
    • false 关闭此功能
